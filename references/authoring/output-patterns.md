# 输出模式

当 skill 需要稳定输出某种结构时，用模板、范式和示例约束结果质量。

## 严格模板

适用于 API、JSON、固定报告格式或有强约束的文档。

```md
## 输出模板

始终使用以下结构：

# 标题
## 摘要
## 关键发现
## 建议
```

## 柔性模板

适用于需要保持整体风格，但允许按上下文增删小节的任务。

```md
## 默认结构

- 背景
- 结论
- 风险
- 下一步

如任务更适合其他组织方式，可在保持可读性的前提下调整。
```

## 输入/输出示例

当 style、格式或粒度很关键时，示例往往比抽象描述更有效。

```md
示例：
输入：请生成一个 skill 融合方案
输出：先给摘要，再给关键改动、测试计划和假设
```

## 何时不要加模板

以下情况不要强塞模板：

- 任务本身探索性很强
- 输出类型高度多样
- 模板会掩盖真正重要的判断过程
