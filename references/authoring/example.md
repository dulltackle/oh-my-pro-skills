# Writing Release Notes

## 使用方式

1. 先识别版本范围和受众
2. 提炼用户可感知变化
3. 单独列出兼容性、迁移和风险
4. 最后再补技术细节

## 顶层应保留的内容

- 什么时候使用这个 skill
- 输出结构
- 哪些内容必须出现
- 哪些细节要去 `references/` 查

## 下沉到 `references/` 的内容

- 不同发布渠道的格式差异
- 历史版本示例
- 术语表
- 公司内部发布规范

## 示例

用户说：“帮我写这个版本的发布说明，重点写升级风险和迁移方式。”

skill 应该：

1. 先收集版本范围和关键变更
2. 先写用户影响，再写技术实现
3. 明确列出风险、回滚与迁移动作
