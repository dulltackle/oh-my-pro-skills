# Query: 查询与知识沉淀

> 目录：[目标](#目标) → [流程](#流程) → [导航策略](#导航策略) → [边界](#边界) → [🔒 检查清单](#-query-检查清单)

## 目标

基于已编译的 Wiki 回答问题——**更重要的是：把有价值的问答结果写回 Wiki，让消费过程也变成生产过程**。

> 这是整个系统最被低估的一个洞察：一次好的查询产生的对比分析、关联发现、综合结论，和摄入一篇新文章一样有价值。如果这些只留在聊天记录里，就是浪费。

## 流程

### Step 1: 理解问题类型

| 类型 | 示例 | 定位策略 |
|------|------|---------|
| 事实型 | "XX是什么？" | 概念页 → 术语表 |
| 对比型 | "A和B的区别？" | 两个概念页 + 关系图 |
| 综合型 | "XX领域的发展趋势？" | 多个专题页 + 交叉分析 |
| 开放型 | "你怎么看XX？" | 基于知识库的有来源支撑的观点 |

### Step 2: 定位相关知识（按优先级）

通过 provider 接口定位：

1. 通过 `wiki_read` 读取 **`wiki/index`** → 快速扫描摘要定位相关页面（中等规模下首选，不需要向量 RAG）
2. 通过 `wiki_read` 读取 **`wiki/glossary`** → 定义类问题
3. 通过 `wiki_list(prefix=concepts)` + `wiki_read` → 核心定义和要点
4. 通过 `wiki_list(prefix=topics)` + `wiki_read` → 跨文档综合视角
5. 通过 `wiki_list(prefix=summaries)` + `wiki_read` → 需要更多细节时深入
6. 通过 `raw_read` → 仅在 Wiki 信息不足时回溯（应该很少发生）
7. 如果 provider 支持 `search` 接口，可直接全文搜索；不支持时回退到 `wiki_list` + 逐页 `wiki_read`

### Step 3: 综合回答

```markdown
## 回答
<基于 Wiki 的综合分析>

## 来源
- [[概念A]](wiki/concepts/A) · [专题B](wiki/topics/B) · 摘要: <标题>(wiki/summaries/xxx)

## 相关延伸
- 可能还感兴趣: [[概念C]]、[专题D]
```

### Step 4: 写回 Wiki ⭐ — 核心步骤

每次 Query 后必须评估：这次问答是否产生了**新的、可复用的知识**？

| 产出类型 | 判断标准 | 写回目标（逻辑路径） |
|---------|---------|-------------------|
| 新的综合洞察 | 跨页面的新结论 | 追加到对应 `wiki/topics/<topic>` 的"关键发现" |
| 概念间新关联 | 之前未记录的联系 | 更新 `wiki/links/relations` |
| 对比分析 | A vs B 的结构化对比 | 新建或更新 `wiki/comparisons/<name>` |
| 常见问题的高质量回答 | 用户可能再问的问题 | 新建或更新 FAQ 页 |
| 知识缺口发现 | "Wiki 里没有但应该有" | 记录到 `wiki/log` 作为待补充 |

**写回规则**：
- 通过 `wiki_read` 读取现有内容，以追加或插入方式写入，不覆盖已有内容，再通过 `wiki_write` 写回
- 更新 `wiki/index` 和反向链接
- 在 `wiki/log` 中记录操作（格式见 `references/compile.md` 的 log canonical 格式）
- 如果只是临时性问答（天气类、简单事实确认），不写回

## 导航策略（按规模）

| 规模 | 推荐策略 |
|------|---------|
| 小 (< 30 页) | `wiki_list(prefix=summaries)` 扫描标题 |
| 中 (30~300 页) | **从 `wiki/index` 开始**，效果出奇地好 |
| 大 (300+ 页) | `wiki/index` + provider 的 `search` 接口 |

## 边界

- Wiki 中没有的信息 → 明确说"暂无"，并建议补充方向
- 每个关键论断标注来源
- 发现薄弱方向时主动建议用户补充素材

## 🔒 Query 检查清单

- [ ] 已通过 `wiki_read` 读取 `wiki/index` 定位相关页面
- [ ] 优质答案如具有沉淀价值，**已通过 `wiki_write` 写回 Wiki**
