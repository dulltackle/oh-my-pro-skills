# 验证检查

构建完成后，用这些检查确认界面没有退回默认值，也没有破坏基础工艺。

## 间距

所有间距值是否都遵循同一个基础单位？

- 图标间隙是否一致？
- 组件内部间距是否成体系？
- 区块之间的距离是否能体现关系远近？
- 是否存在无法解释的随机数值？

## 深度

深度策略是否前后一致？

- 如果采用仅边框策略，是否避免了阴影？
- 如果采用阴影策略，阴影是否微妙且层级清晰？
- 表面层级是否可以在眯眼时被感知？
- 下拉菜单、弹出层是否明显位于触发来源之上？

## 颜色

颜色是否都服务于结构、状态、动作、强调或产品性格？

- 是否存在无意义的装饰色？
- 语义色是否只用于语义场景？
- 表面是否保持同一色相，只通过明度建立层级？
- 文本颜色是否有主要、次要、三级、弱化的层级？

## 组件

重复出现的组件是否保持一致？

- 按钮高度、圆角、内边距是否一致？
- 输入框、选择器、筛选器是否共享同一控件语言？
- 卡片表面处理是否一致，同时内部结构是否针对内容设计？
- 交互元素是否具备默认、悬停、按下、焦点、禁用状态？

## 内容

界面内容是否像真实产品中的真实数据？

- 标题、指标、表格和导航是否讲同一个故事？
- 数据是否具体，避免占位式泛化？
- 用户下一步要做什么是否清晰？
- 空状态、加载状态和错误状态是否可理解？
