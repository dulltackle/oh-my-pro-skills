## 沉淀协议

**触发**：用户输入"请沉淀本次生成的经验和教训"

**流程**：回顾 → 分类 → 写入 → 回注提议

1. **回顾**：提取迭代轮次、反复出现的改动（≥2次 = 痛点）、胜出方案参数
2. **分类**：
   - 通用规则 → 提议更新 `anti-patterns.md`，**不写入本文件**
   - 项目经验（领域偏好、素材处理、布局决策理由）→ 写入本文件
3. **写入**：追加记录 + 更新索引表
4. **回注提议**：如有 anti-patterns 候选，向用户确认后再改

**条目格式**：

```
### #NNN：项目名（日期）
- **概况**：主题 / 风格·布局 / 迭代N轮
- **Top 3 教训**：
  1. ...（每条≤2句话）
  2. ...
  3. ...
- **胜出参数**：关键尺寸·配色（如有参考价值）
```

## 详细记录

### #001：刘丰硕·格律乾坤医学海报（2026-04-10）

- **概况**：青少年抑郁症专题线上交流会讲座海报 / Organic Natural（鼠尾草绿+暖米白）/ Layout A 照片主角型 / ~12 轮
- **Top 3 教训**：
  1. 首版必须做完整字号审计表（Hero:Body ≥2.5:1，同级差 <1.15x），反复调字号占返工 40%（通用规则见 `anti-patterns.md` #5）
  2. 透明底 PNG 用 `object-fit:contain` + `drop-shadow`，不是 cover；写代码前用 file 检查 RGBA 通道，避免当黑底处理浪费多轮
  3. "突出 XX"按 4 级升级（加粗→改色→胶囊→强对比），先问用户这是第几重要的信息来决定起始级别
- **胜出参数**：鼠尾草绿+暖米白 / 标题 82-100px / 正文 28-34px

### #002：格律乾坤·青少年抑郁症专题海报（2026-04-12）

- **概况**：同一项目第二轮优化 / 靛蓝紫粉渐变浅底 / 上下分区杂志式 / 9 轮
- **Top 3 教训**：
  1. 照片宽度 370-420px、高度不超过标题区 90%；一味放大照片会挤压标题导致失衡
  2. 嘉宾标签风格跟海报调性绑定：亮底→白底轻标签；深底→实心重标签，混搭必翻车
  3. 每次改动后必须截图视觉检查，连续改多处不验证容易过犹不及（第 9 版同时改照片+标签，不如第 8 版）
- **胜出参数**：1080×1320 / #f5f3ff→#fef3c7 / 标题 96px 渐变+52px 副标题 / 照片 400×490 contain / 紫色实心时间卡 40px 白字
