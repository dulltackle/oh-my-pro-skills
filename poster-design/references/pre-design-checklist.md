# 预设计决策清单（写代码前必做）

> 来自实战教训：**80% 的海报问题在设计阶段就已注定，不是实现阶段。**
> 写第一行 CSS 之前，先在这里把决策做完。

## 为什么需要这个阶段

没有预设计的典型后果：
- 照片截断 / 溢出 / 比例失调
- 7-8 个模块互相抢空间，上下比例失衡
- 为了填充空白加空洞卡片（形式 > 内容）
- 出图后才发现明显问题，反复返工

## 第一步：信息审计与优先级排序

### 列出所有信息项

把用户提供的所有内容逐条列出来，每条标注类型：

| 类型 | 示例 | 典型处理方式 |
|------|------|-------------|
| 人物姓名 | 孙逊 | Hero 级，最大字号 |
| 头衔/身份 | 北京中医药大学教授 | 标签组或列表，2-4 条精选 |
| 照片 | 人物肖像 | 视觉锚点，需确认比例 |
| 核心成就 | 15项专利、百篇论文 | 高亮数字或 badge |
| 专业描述 | 一段介绍文字 | 内容卡主体 |
| 装饰性标签 | 世界首创、泰斗 | 视情况合并到其他区域 |
| 机构/来源 | 底部 footer | 最小化 |

### 做减法：三级分类

```
🔴 必须突出（1-3 项）：姓名 + 照片 + 一句话定位
🟡 重要但不抢眼（2-4 项）：核心头衔 + 关键成就数字
⚪ 可以弱化或合并（其余）：次要头衔合为一行、装饰性标签融入其他区
```

**硬规则：如果 ⚪ 级项目超过 5 个，必须砍或合。**

### 常见的信息冗余

以下情况说明有冗余，需要合并：
- 成就卡片的内容和专业擅长文字说的是同一件事 → 去掉卡片，保留文字
- 头衔列表超过 5 条 → 合并同类项（如"北京中医药大学教授 / 北京高等中医药培训学校教授"可合并为"北京中医药大学及培训学校教授"）
- 同一数字在 2 个地方出现（如"15项专利"既在 badge 又在正文）→ 只保留一处

## 第二步：空间预算分配

### 先画草图分配占比

在写任何 HTML 之前，先用百分比规划各区域占画布高度的比例：

```
标准竖版海报（1080×1920）空间预算模板：

┌─────────────────────┐
│  Brand Row     ~2%   │  固定极小
├─────────────────────┤
│  Title/Hero    ~15%  │  记忆点，紧凑有力
├─────────────────────┤
│  照片+人物信息  ~35%  │  视觉锚点，最大区块
├─────────────────────┤
│  核心内容       ~25%  │  正文/描述，呼吸感
├─────────────────────┤
│  成就/亮点      ~12%  │  精炼，不贪多
├─────────────────────┤
│  Footer        ~6%   │  极简
├─────────────────────┤
│  自然留白       ~5%   │  不是 bug，是设计
└─────────────────────┘
```

### 预算校验规则

1. **照片+人物信息区 ≤ 45%** — 超过这个比例，下面必然被压缩
2. **Title/Hero 区 ≤ 18%** — 标题再大也不该吃掉近 1/5 的画布
3. **每个内容区块至少保留 10%** — 低于这个值文字会密不透风
4. **总预留留白 3-8%** — 不要把 1920px 填满到像素级

### 如果内容多放不下？

按优先级砍，而不是缩小所有字号硬塞：
1. 先砍 ⚪ 级项目
2. 再合并同类头衔
3. 再缩短正文（保留关键句）
4. 最后才考虑微调字号（±2px 级别）

### ⭐ 字号优先原则

1. **先定正文 ≥ 28px**，再反推标题大小
2. **宁可缩标题/照片，不压正文**
3. 辅助文字 ≥ 16px
4. 标题:正文 ≥ 2.5:1（人物介绍可放宽，纯活动海报保持 ≥ 4:1）

→ 完整尺度体系和字号审计方法见 `typography.md`
→ 空间预算计算用 `scripts/space_budget.py --layout X`

## 第三步：素材适配检查

有照片时，必须按 `image-checklist.md` 完整执行素材检查。无照片时跳过本步。

裁剪技巧和 CSS 融入策略见 `image-cropping.md`。

其他素材注意：
- **Logo**：确认格式（PNG/SVG）和透明背景
- **二维码**：预留正方形区域，边长 ≥ 120px@2x
- **图标/插画**：确认风格统一（线面/填充/扁平不能混）

---

## 第四步：视觉动线预演

### 3 秒浏览路径模拟

在脑子里（或草稿纸上）走一遍用户的视线路径：

```
第 1 秒：眼睛落在哪里？→ 应该是姓名 or 大标题
第 2 秒：视线移向哪里？→ 应该是照片（视觉锚点）
第 3 秒：然后呢？→ 应该是核心成就/专业描述
最后：   CTA or Footer
```

### 动线断裂检测

常见断裂信号：
- 两个重要元素之间有大片空白 → 加过渡元素（分隔线/小标签）
- 视线需要"跳过"某个区域才能到下一个重点 → 调整顺序
- 三个以上元素争夺"第一眼" → 砍掉竞争者，只留一个

① 所有信息列出来了吗？

③ 各区域空间预算分配了吗？照片区 ≤ 45%？
④ 照片实际比例确认了吗？容器匹配吗？
⑤ 3秒动线模拟过了吗？有断裂吗？

任何一个 No → 先解决再动手。
