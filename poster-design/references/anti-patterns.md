# 反模式清单

来自 15 轮海报迭代 + 3 个外部设计 Skill 的血泪总结。

## 🚨 致命级（必须避免）

| # | 反模式 | 为什么 | 正确做法 |
|---|--------|--------|---------|
| 1 | **底部大段空白** | `margin-top:auto` 把 footer 推到底部，内容只有 1400px 但画布 1920px | 去掉 flex-col+h-full，让内容自然流动；或放大核心元素 |
| 2 | **CTA 不突出** | 报名信息和标题/照片一样平淡，用户不知道该做什么 | CTA 用**全页最强色**（渐变背景+脉冲光晕+最大字号） |
| 3 | **文字对比度不足** | 页脚 opacity <0.4 或浅色字在浅底上 | 正文 ≥4.5:1 对比度；页脚 ≥0.5 opacity；用深色而非浅色 |
| 4 | **AI slop 配色** | 紫白渐变、彩虹色、三卡片+蓝紫主色 | 先选设计方向，再从方向对应的调色板选色 |
| 5 | **标题巨大、其余全小字** | 主标题100px+但正文<24px、头衔<16px、Footer<13px——**信息传递失效** | 先定正文字号≥28-30px再反推标题；宁可标题82px配30px正文，不要100px配22px |
| 6 | **下半部文字隐形** | 卡片描述<16px+opacity<0.75、Footer像水印、badge看不清 | 卡片描述≥20px opacity≥0.85；Footer≥15px opacity≥0.7；**出图审查D5专项检查** |

## ⚠️ 严重级（明显降低质量）

| # | 反模式 | 为什么 | 正确做法 |
|---|--------|--------|---------|
| 7 | **标题不够大** | Hero title <80px，和正文没有戏剧性跳跃 | Title ≥82px（人物介绍）或≥96px（活动海报），body ≥28px，比例 ≥2.5:1 |
| 8 | **Emoji 当图标** | 👩‍⚕️💆🏥📚 在各平台渲染不一致，不专业 | 用 ✦ 符号、SVG 图标、或纯文字 |
| 9 | **间距随机混用** | margin 7px, 13px, 21px 随意取值 | 统一 Tailwind scale：4/8/12/16/20/24/32 |
| 10 | **所有卡片长得一样** | 内容卡、info卡、audience 卡全是白底圆角 | 变化 bg 透明度、border 样式、内边距 |
| 11 | **纯色扁平背景** | 一片 #fff 或 #f5f5f5，没有任何深度 | 径向渐变 mesh + 噪点纹理 + 大形状光斑（至少 2 层） |
| 12 | **装饰性填充区** | 内容不够长时加 "愉悦·美丽·健康" 竖排文字填充 | 从根本解决——放大元素让内容撑满 |

## 💡 中等级（影响专业感）

| # | 反模式 | 为什么 | 正确做法 |
|---|--------|--------|---------|
| 13 | **居中大段正文** | >2 行的文字居中排列，阅读体验差 | 仅标题/单行短文居中，正文左对齐 |
| 14 | **视觉层级扁平** | 所有元素看起来一样重要，没有主次 | 用字号(4x+跳跃)、颜色强度、位置创造清晰层级 |
| 15 | **字体配对重复** | 每个项目都用 Playfair+Source Sans | 每次从配对池选不同组合 |
| 16 | **footer 信息过多** | slogan + 机构 + 地址 + 电话 + 版权... | 最多 3 行：分隔线 + 机构名 + 一句话 |
| 17 | **照片无处理** | 直接放一张图，无边框无投影无光晕 | 加 glow 背景 + 微边框 + 角装饰 + drop-shadow |
| 18 | **信息不加筛选全堆上** | 用户给的头衔/成就全部展示，6+ 个头衔列表 | 做三级分类（🔴🟡⚪），⚪ 级超 5 个必须砍或合 |
| 19 | **不确认照片比例就定容器** | 用固定 aspect-ratio（如 3:4）套所有图，实际图片是方形 → 截断或留白 | 写代码前 `file` 或目视确认比例，根据实际选容器策略 |
| 20 | **空间预算失衡** | 标题+照片吃了 70% 画布，内容区被压到 30% 密不透风 | 预先分配各区域占比：照片≤45%、标题≤18%、每内容区≥10% |
| 21 | **形式大于内容的填充卡片** | 成就三卡片的内容跟正文完全重复，只为"模板里有这个位置" | 检查每个模块是否有独立信息价值，无则删 |
| 22 | **截图后不自审直接发** | 出现截断/溢出/重叠等肉眼可见问题却没发现 | 截图后必须执行双轨制审查，详见 `references/visual-review.md` |

## 自检清单

### A. 预设计阶段（写代码前）

- [ ] 信息已列出并做了 🔴🟡⚪ 三级分类
- [ ] ⚪ 级项目 ≤ 5 个（超了必须砍或合）
- [ ] 各区域空间预算已分配（照片区 ≤45%，标题 ≤18%）
- [ ] 照片实际宽高比已确认，容器策略已匹配
- [ ] 3 秒视觉动线已预演，无断裂
- [ ] 每个计划中的模块都有独立信息价值（无重复填充）

### B. 实现阶段（写代码时）

- [ ] 设计方向已明确且一致
- [ ] 标题 ≥96px，与正文比 ≥4:1
- [ ] CSS 变量已定义配色（不是硬编码颜色）
- [ ] 强调色只用了 1 个
- [ ] 无 emoji（图标用 SVG 或 ✦）
- [ ] 间距统一在 Tailwind scale
- [ ] CTA 是全页视觉焦点（如有）
- [ ] 底部空白 <100px
- [ ] 页脚文字 opacity ≥0.5
- [ ] 背景有至少 2 层深度（渐变/纹理/形状）
- [ ] 照片有框架处理（glow/border/corners）
- [ ] 字体配对与上次项目不同

### C. 出图后自审（截图后）

出图后必须执行双轨制审查，详见 `references/visual-review.md`。

## 🔧 技术问题（Puppeteer / 工具链）

### 截图阶段问题

| 症状 | 最可能原因 | 修复 |
|------|-----------|------|
| `Failed to launch browser` | Chrome 未安装或路径不对 | `which google-chrome \|\| which chromium`；指定 `executablePath` |
| 截图空白/全白 | 字体未加载 / CSS 变量未定义 | 加 `waitUntil:'networkidle0'`；检查 `:root` 块完整性 |
| 文字乱码/方框 | Google Fonts 未加载（CDN 被墙） | 系统字体回退 `'Noto Serif SC','SimSun',serif` 或本地 `@font-face` |
| 图片加载失败(404) | 路径错误 | 写代码前 `ls -la` 确认路径存在 |

### 性能与文件管理

- HTML 正常范围 < 50KB，超标时检查内联 base64 和重复 CDN 引入
- 文件命名：`poster-{主题}-v1.html` → `v2.html` → `final.html` → `.png`
