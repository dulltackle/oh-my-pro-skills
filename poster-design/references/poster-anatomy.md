# 海报区块角色定义

> **本文件只回答"每个区块是什么、为什么存在、有什么职责"。**
> 代码实现见 `scaffold.py --layout X`（按布局模式生成完整骨架）。
> 布局选择见 `layout-patterns.md`。

## 区块总览

```
┌─────────────────────┐
│  ① Brand Row        │  身份标记，极轻量
├─────────────────────┤
│  ② Title Area       │  THE 记忆点，3秒内必须被记住
├─────────────────────┤
│  ③ Hero Zone        │  视觉锚点（通常是照片）
├─────────────────────┤
│  ④ Tags             │  3秒扫读关键词
├─────────────────────┤
│  ⑤ Content Card     │  核心信息主体
├─────────────────────┤
│  ⑥ Info / Audience  │  实用信息（时间/地点/人群）
├─────────────────────┤
│  ⑦ CTA + Footer     │  行动召唤 + 收尾
└─────────────────────┘
```

**注意：不是每张海报都需要全部 7 个区块。** 人物介绍海报可能没有 CTA 和 Info Bar；极简风格可能只有 ①②③⑦。根据实际需求取舍。

**总内容高度目标：1700-1850px**（留 80-220px 给自然底部间距）

---

## ① Brand Row（品牌行）

**角色**：建立信任背书，不抢注意力。

| 属性 | 要求 |
|------|------|
| 字号 | ≤ 13px |
| opacity | 0.4-0.55 |
| 风格 | 品牌 uppercase + wide tracking；日期斜体衬线 |
| 高度 | ~30px，固定极小 |

**❌ 不要**放 logo 大图或 slogan。

---

## ② Title Area（标题区）⭐ THE MEMORABLE ELEMENT

**角色**：整张海报的记忆点。3 秒内必须被记住。

### 尺度要求

| 元素 | 最小 | 推荐 | 最大 |
|------|------|------|------|
| 主标题 | 72px | **96-112px** | 130px |
| 副标题 | 80px | **100-120px** | 140px |

**关键比例：主标题 : 正文 ≥ 4:1**

### 装饰要素

标题下方不要留白——加几何装饰元素（长线/点/短线/圆框组合）。
详见 `tailwind-recipes.md` → Layout A2 标题区组件。

---

## ③ Hero Zone（主角区）

**角色**：视觉锚点。通常是主讲人照片，也可以是主视觉图。

### 照片处理要点

| 项目 | 推荐值 | 说明 |
|------|--------|------|
| 宽度 | 480-520px（Layout A）或按布局模式调整 | 占画幅 ~45-50% |
| object-fit | contain 或 cover（取决于裁剪策略） | 详见 `image-cropping.md` |
| 边框/投影 | 必须有 | 至少 2 项：border + box-shadow |
| 装饰边框 | 推荐 | 角装饰 / 微边框 / 光晕背景 |

### 人物信息列（照片旁）

- 姓名：大字号衬线（52-72px），带下划线装饰
- 拼音：斜体西文，opacity 0.4
- 头衔：竖排或横排列表，控制数量（≤5 条，多则合并）

---

## ④ Tags（标签栏）

**角色**：3 秒扫读的关键词。

| 属性 | 要求 |
|------|------|
| 数量 | 3-5 个，不多不少 |
| 形状 | rounded-full（胶囊形）> rounded（矩形） |
| bg | accent 的 8-12% 透明度 |
| border | accent 的 16-20% 透明度 |
| 图标 | 不用 emoji，用 ✦ 或纯文字 |

---

## ⑤ Content Card（内容卡）

**角色**：核心信息列表（课程大纲/活动亮点/专业描述）。

| 属性 | 要求 |
|------|------|
| 背景 | 半透明白色（white/70%）+ 微边框 |
| 内边距 | px-8 py-5 左右 |
| 投影 | 柔和 shadow + inset 高光 |
| 列表 | 左侧竖线引导视线 + 勾选图标 |
| 字号 | 24px（比网页正文粗，海报阅读距离远） |

---

## ⑥ Info Bar + Audience

### Info Bar

**角色**：时间 / 地点 / 报名等实用信息。

| 属性 | 要求 |
|------|------|
| 布局 | 横向等分（2-3 列）|
| 每列结构 | 小标签(uppercase) + 大数字/文字 |
| 分隔 | 右侧细线分隔（最后一列无）|
| 背景 | 更浅的半透明白色 |

**报名一栏的视觉权重应略高于时间和地点。**

### Audience Grid

**角色**：适合人群快速展示。

| 属性 | 要求 |
|------|------|
| 列数 | 4 列网格 |
| 图标 | 统一用 ✦ 或 SVG |
| 文字 | 18px，紧凑行高 |

---

## ⑦ CTA + Footer

### CTA ⭐ 第二重要区块

**CTA 是全页视觉焦点——比标题更强调行动。**

| 属性 | 要求 |
|------|------|
| 背景 | 渐变色（accent-deep → accent）|
| 文字 | 白色，大字号（34px+）|
| 装饰 | 点阵纹理 overlay + 光晕 shadow |
| 按钮 | 圆角胶囊，内嵌箭头图标 |

### Footer

**角色**：收尾，极简。

| 属性 | 要求 |
|------|------|
| 行数 | 最多 3 行 |
| 结构 | 分隔线 + 机构名 + 一句话 |
| opacity | 0.35-0.6 |
| **不要** | slogan + 地址 + 电话 + 版权堆一堆 |
