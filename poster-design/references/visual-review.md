# 出图视觉审查规范

> Puppeteer 截图后，**必须**执行本审查。这是 poster-design 质量的最后一道防线。
> 本文档定义"怎么看、看什么、怎么判、不通过怎么办"。

## 为什么需要视觉审查

代码层面的检查（DOM 尺寸、z-index 值、overflow 属性）只能发现**一部分问题**。

以下问题**只有看到渲染结果才能发现**：

| 问题类型 | DOM 检查能发现？ | 视觉检查能发现？ |
|---------|-----------------|-----------------|
| 照片人物被截断（头/手/身体） | ❌ 不知道图片内容 | ✅ 一眼看出 |
| 照片裁切位置尴尬（如刚好切在手腕） | ❌ 只知道用了 cover | ✅ 看到具体位置 |
| 文字与背景对比度不够 | ⚠️ 能算比值但不知道实际观感 | ✅ 直接感受 |
| 元素之间视觉权重冲突 | ❌ 每个 element 都正常 | ✅ 整体一看就不对 |
| 整体"像 AI 生成的" | ❌ 没有审美判断 | ✅ 综合气质判断 |
| badge/装饰盖住了关键内容 | ⚠️ 有时 z-index 看似正确但渲染异常 | ✅ 实际看到覆盖 |
| 上下比例失衡（一眼感觉） | ❌ 各区 px 数值都合理 | ✅ 视觉比重不对 |

**结论：DOM 检查和视觉检查是互补的，缺一不可。**

---

## 审查流程（双轨制）

### 轨道 A：DOM / 代码层检查（快速，先做）

在截图**之前或同时**做。目标是拦截低级错误。

```bash
# 可选：用 Puppeteer eval 快速检查
page.evaluate(() => {
  const issues = [];
  // 1. 检查文字溢出
  document.querySelectorAll('p, div, span').forEach(el => {
    if (el.scrollWidth > el.clientWidth + 1) {
      issues.push(`溢出: ${el.className?.substring(0,40)} scrollW=${el.scrollWidth} clientW=${el.clientWidth}`);
    }
  });
  // 2. 检查图片加载
  document.querySelectorAll('img').forEach(img => {
    if (!img.complete || img.naturalHeight === 0) {
      issues.push(`图片未加载: ${img.src}`);
    }
  });
  return issues;
});
```

**A 轨通过标准：无溢出、图片全部加载成功。**

### 轨道 B：视觉层审查（核心，必须做）

截图后，用 `read` 工具读取 PNG 图片文件，触发视觉能力，按以下 **5 维度 × 3 级评分** 逐项审查。

---

## 视觉审查 5 维度

### 维度 1：完整性（Completeness）

**问自己：所有该显示的内容都完整显示了么？**

| 检查项 | 通过标准 | 不通过示例 |
|--------|---------|-----------|
| 人物照片 | 面部完整、身体自然截断（不在关节处）、无重要部分缺失 | 头被切、下巴没了、手臂截断在肘部 |
| 文字信息 | 所有文字在画面内、无被遮挡的文字 | 标题跑出边界、长头衔被截断 |
| 关键元素 | Logo/badge/标签完整可见 | badge 只露一半、footer 被截 |

**评分：✅ 通过 / ⚠️ 轻微（不影响理解）/ ❌ 不通过**

### 维度 2：比例与空间（Proportion & Space）

**问自己：各区域占的视觉比重合理吗？**

| 检查项 | 通过标准 | 不通过示例 |
|--------|---------|-----------|
| 上下平衡 | 没有"上半极满+下半极空"或相反的感觉 | 标题+照片吃了 70%，内容挤在 30% |
| 照片占比 | 照片区域占画布 40-50%（Layout A） | 照片大到挤压所有其他元素 |
| 底部空白 | footer 下方留白 ≤100px，不是悬空也不是紧贴 | footer 孤零零浮在中间，下方大片白 |
| 区块呼吸 | 每个区块之间有适当间距，不拥挤不稀疏 | 所有东西挤在一起，或者散得找不到关系 |

**参考：Step 1 的空间预算分配——如果视觉上觉得不对，大概率是预算没遵守。**

### 维度 3：层级与焦点（Hierarchy & Focus）

**问自己：3 秒内能看出重点是什么吗？**

按以下顺序模拟视线：

```
第 1 秒 → 最醒目的是什么？（应该是标题 or 姓名）
第 2 秒 → 视线移向哪里？（应该是照片 or 核心信息）
第 3 秒 → 注意到什么？（应该是关键描述/亮点）
最后     → 有行动指引吗？（如有 CTA）
```

| 检查项 | 通过标准 | 不通过示例 |
|--------|---------|-----------|
| 第一眼 | 明确落在主标题或人物姓名上 | 不知道该看哪，多个元素抢焦点 |
| 视觉路径 | 流畅自然，没有跳跃感 | 看完标题不知道往哪看 |
| 主次分明 | 大中小层级清晰（≥4x 跳跃） | 所有字差不多大，扁平 |
| CTA（如有） | 是全页最想点击的地方 | CTA 跟普通文字没区别 |

### 维度 4：协调性与质感（Harmony & Quality）

**问自己：这张海报看起来像一个精心设计的整体，还是拼凑的？**

| 检查项 | 通过标准 | 不通过示例 |
|--------|---------|-----------|
| 配色统一 | 全案只有一个强调色，色调一致 | 突然出现异色元素 |
| 字体一致 | 标题衬线+正文无衬线，配对有逻辑 | 三种以上字体混用 |
| 照片处理 | 有边框/投影/光晕至少 2 项处理 | 照片直接贴上去，生硬矩形 |
| 背景深度 | 有渐变 mesh 或纹理，不是纯色平涂 | 一片死白 #fff 或 #f5f5f5 |
| AI 感 | 不像 AI 生成（无紫白渐变、无彩虹、无 emoji 图标） | 一看就是 AI slop |

### 维度 5：细节精度（Detail Precision）

**问自己：凑近看有没有粗糙的地方？**

| 检查项 | 通过标准 | 不通过示例 |
|--------|---------|-----------|
| 对齐 | 左右边缘对齐、元素间距均匀 | 歪的、间距忽大忽小 |
| 边距 | 符合 Tailwind scale（4/8/12/16/20/24/32） | 出现 7px、13px 这种随机值 |
| 装饰元素 | 角标/分隔线/圆点位置精确 | 装饰歪了、大小不一 |
| 文字质量 | 无乱码、无方框、字体渲染正常 | 字体没加载成 fallback |
| 特殊元素 | badge/标签不盖住主要内容 | badge 叠在照片脸部上 |
| **下半部文字** | 卡片描述≥18px、Footer≥15px、opacity≥0.7 | 卡片描述<16px且opacity<0.8、Footer像水印 |

## 评分与决策矩阵

### 逐维度打分

每个维度评一级：✅ 通过 / ⚠️ 轻微问题 / ❌ 严重问题

### 决策规则

| 结果 | 条件 | 行动 |
|------|------|------|
| **🟢 放行** | 5 个维度全 ✅，或仅 1-2 个 ⚠️ 且不影响核心信息传递 | 发送给用户 |
| **🟡 小修后放行** | 有 ⚠️ 但修复量 < 5 分钟（调个 padding、改个字号、移动下位置） | 修复 → 重新截图 → 仅复查修改涉及的维度 |
| **🔴 返工** | 任一维度 ❌，或 ⚠️ 超过 3 个 | 回到对应 Step 重新决策/实现，修完后重新走完全部审查 |

### 常见 ⚠️ 问题快速修复指南

| 问题 | 修复方式 | 改哪个参数 |
|------|---------|-----------|
| 照片稍微有点大/小 | 调整容器 width 或 height | ±20~40px |
| 标题字号稍大/稍小 | 调整 font-size | ±4~8px |
| 两元素间距不舒服 | 调整 gap/margin/padding | ±8~16px |
| 底部空白略多/少 | 调整内容区 padding-bottom 或某区块 margin | ±16~32px |
| 某处颜色稍微跳脱 | 微调 opacity 或换同色系深浅 | opacity ±0.1 |
| badge 位置微偏 | 调整 position top/left | ±8~16px |
