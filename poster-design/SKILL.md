# Poster Design Skill

用 HTML + CSS（优先 Tailwind）+ Puppeteer 截图，产出**不像 AI 生成**的高质量宣传海报。

## 核心哲学

**一个观点，一个记忆点。** 3 秒内让观者明白：这是什么、为什么重要、该做什么。

**布局做 80% 的工作，颜色和字体做 20%。** 布局失衡是海报失败的第一原因。

**决策先于实现。** 80% 的问题在设计阶段就已注定（信息过载、空间失衡、素材不适配）。

**文字是信息传递的主途径。** 先定正文字号（≥28px），再反推标题大小。

## 判断框架

拿到需求后，先沿这 3 个维度判断，而不是直接进入固定流程：

### 维度 1：任务类型

| 类型 | 特征 | 下一步 |
|------|------|--------|
| **全新设计** | 从零开始做海报 | 走 `workflow.md` 完整流程 |
| **修改现有海报** | 已有 HTML/PNG，用户不满意 | 查资源路由表定位问题 → 对应 reference 修复 |
| **调参优化** | 微调字号/间距/配色 | 直接查对应 reference |

### 维度 2：信息复杂度

| 复杂度 | 特征 | 推荐布局 |
|--------|------|---------|
| 简单（≤5 条核心信息）| 无照片或极少文字 | Layout B 居中 / Layout D 全幅冲击 |
| 中等（5-10 条）| 有照片+文字+标签 | Layout A 照片主角 |
| 复杂（10+ 条）| 多模块、多主题 | Layout C 分割杂志 |

### 维度 3：风险点

| 风险信号 | 应对 |
|---------|------|
| 信息过载（>10 条不可砍的信息）| 强制执行信息审计 |
| 照片比例未知 | 写代码前必须完成素材检查 |
| 用户对视觉质量要求高 | 出图后必须执行双轨审查 |
| 不确定设计方向 | 默认用 Warm Editorial |

## 技术栈

| 层 | 技术 |
|----|------|
| 模板 | HTML + Tailwind CSS (CDN) |
| 出图 | Puppeteer |

## 🔒 关键硬规则（不可违反）

### 设计质量规则

1. 正文 ≥ 28px，头衔/标签 ≥ 16px，Footer ≥ 15px / opacity ≥ 0.7
2. 照片区 ≤ 45%，标题区 ≤ 18%
3. 只 1 个强调色，背景 ≥ 2 层深度
4. 禁止 emoji 图标、禁止纯黑 `#000` / 纯白 `#fff`
→ 详细解释和违规示例见 `anti-patterns.md`

### 执行纪律

5. 首版设计：必须按完整流程执行（含风险分级判断）
6. 每一版出图：必须经过双轨审查后才能发送
7. 图片素材：写代码前必须完成素材检查

→ 具体读哪些文件、何时读，见下方资源路由表。

| 脚本 | 用途 |
|------|------|
| `scripts/scaffold.py --layout A` | 生成 HTML 骨架（加 `--snippet A5` 输出单个组件） |

## 输入输出

### 用户应提供
- 海报主题/用途
- 核心文字信息（姓名、头衔、描述、时间地点等）
- 图片素材（如有；没有则说明）

### 产出
- 高清 PNG（**必须经过视觉能力检查后发出**）
- 可选：HTML 源文件（方便后续修改）
